# CSS Naming Convention

Методологія іменування класів для Webflow-проєктів команди.

Базовий принцип: **базовий клас тримає структуру та спільні стилі, комбо-клас `is-` додає варіацію або стан.** Клас існує лише тоді, коли несе власні стилі; типографіку задають токени `text-size-*` / `text-color-*`.

## Швидка довідка

| Що               | Як іменуємо                       | Приклад                                 |
| ---------------- | --------------------------------- | --------------------------------------- |
| Структурний блок | одне слово / kebab-case           | `section`, `job-card`                   |
| Елемент блоку    | `блок_елемент` (має власні стилі) | `job-card_header`                       |
| Варіація / стан  | базовий клас + `is-`              | `section is-hero`, `card is-featured`   |
| Утиліта          | назва за CSS-властивістю          | `text-align-center`, `margin-top-large` |
| Розмір тексту    | `text-size-*`                     | `text-size-m`, `text-size-h1`           |
| Колір тексту     | `text-color-*` (комбо)            | `text-color-black`                      |

**Логіка розрізнення:**

- `is-` → «цей елемент **є таким різновидом** або **у такому стані**»
- утиліта (`text-align-center`, `margin-top-large`) → «накинь оцю **одну конкретну властивість**»
- токени (`text-size-*`, `text-color-*`) → вигляд тексту, без окремого `_`-класу

## 1. Секції

Усі секції мають один базовий клас `section`. Він тримає спільні властивості: `padding`, `position`, `overflow`. Специфіку конкретної секції додає комбо-клас `is-`.

```html
<section class="section is-hero">
  <section class="section is-vacancies">
    <section class="section is-contact"></section>
  </section>
</section>
```

| Клас           | Призначення                                      |
| -------------- | ------------------------------------------------ |
| `section`      | базові відступи й позиціонування для всіх секцій |
| `is-hero`      | висота, фон, вирівнювання героя                  |
| `is-vacancies` | специфіка секції вакансій                        |
| `is-contact`   | специфіка секції контактів                       |

---

## 2. Компоненти

Той самий принцип: базовий клас = структура й спільні стилі, `is-` = варіація.

```html
<a class="button is-primary">Надіслати</a>
<a class="button is-secondary">Скасувати</a>
<div class="card is-featured">…</div>
```

| Базовий  | Варіації                                                |
| -------- | ------------------------------------------------------- |
| `button` | `is-primary`, `is-secondary`, `is-large`, `is-disabled` |
| `card`   | `is-featured`, `is-horizontal`                          |

---

## 3. Елементи блоку

`_`-клас отримує елемент, що має **власні стилі, специфічні для цього блоку** — найчастіше це layout-обгортки (flex/grid, gap, padding) або елемент із власним виглядом, якого не дає жоден токен. Іменуємо через `_`: `блок_елемент`.

```
job-card           ← блок: вигляд картки (фон, рамка, padding, radius)
job-card_header    ← layout-обгортка: flex-колонка, gap
job-card_tags      ← layout-обгортка: flex-wrap, gap
job-card_tag       ← елемент із власним виглядом: фон, padding, radius
```

### Коли `_`-клас НЕ потрібен

Якщо елемент — це просто текст, і весь його вигляд задають токени `text-size-*` / `text-color-*`, то окремий `_`-клас був би **порожнім** — не створюємо його. Пишемо токени напряму:

```html
<!-- ✅ вигляд тексту повністю описаний токенами -->
<h3 class="text-size-h4 text-color-black">Frontend Developer</h3>

<!-- ❌ job-card_title нічого не тримає й дублює роль токенів -->
<h3 class="job-card_title text-size-h4 text-color-black">Frontend Developer</h3>
```

> **Головне правило: не створюй клас, який не несе власних стилів.**
>
> - Потрібен лише відступ → утиліта (`margin-bottom-s`), а не елемент-клас.
> - Потрібен хук для JS чи таргетинг → атрибут (`data-*`) або окремий цільовий клас, але не порожній стильовий.

> Webflow: уникаємо подвійного підкреслення `__` — лише одинарне `_`. У Style Manager класи групуються логічно.

## 4. Варіації та стани (`is-`)

`is-` читається як стан або різновид — це робить розмітку самодокументованою.

```html
<nav class="nav is-scrolled">
  <!-- навбар після скролу -->
  <div class="modal is-open">
    <!-- відкрита модалка -->
    <button class="tab is-active">
      <!-- активна вкладка -->
      <input class="input is-error" />
      <!-- поле з помилкою -->
      <a class="link is-current"> <!-- поточне посилання --></a>
    </button>
  </div>
</nav>
```

`is-` працює і на елементах блоку:

```html
<div class="job-card_tag is-remote">Remote</div>
```

---

## 5. Утиліти

Утиліта — це **не** варіація, а одна конкретна властивість. Іменуємо її за CSS-властивістю та значенням, без префікса. Навішується самостійним класом, не через `is-`.

```
text-align-center
margin-top-large
hidden-mobile
display-flex
gap-medium
```

```html
<div class="job-card margin-top-large"></div>
```

> Не плутати: `card is-featured` — це різновид картки; `card margin-top-large` — це накинута властивість відступу. Якщо властивість одна й перевикористовується скрізь — це утиліта.

## 6. Текст і типографіка

Текст описуємо двома сімействами класів: **розмір** (`text-size-*`) і **колір** (`text-color-*`). Розмір — базовий клас, колір — комбо-клас поверх нього. Окремий `_`-клас тексту **не** потрібен (див. §3).

### Розміри

Шкала для звичайного тексту:

```
text-size-xs
text-size-s
text-size-m
text-size-l
text-size-xl
```

Заголовкові розміри — окремо, відв'язані від семантичного тегу:

```
text-size-h1
text-size-h2
text-size-h3
text-size-h4
text-size-h5
text-size-h6
```

> **Важливо:** `text-size-h1` — це **візуальний розмір**, а не HTML-тег. Семантику (`<h1>`, `<h2>`…) обираємо за структурою сторінки й SEO, а вигляд — класом `text-size-h*`. Тож на `<h2>` можна навісити `text-size-h1`, якщо так вимагає дизайн, не ламаючи семантику.

### Колір

Колір додаємо комбо-класом `text-color-*` поверх розміру:

```html
<h1 class="text-size-h1 text-color-black">Заголовок</h1>
<p class="text-size-m text-color-muted">Опис</p>
```

```
text-color-black
text-color-white
text-color-primary
text-color-secondary
text-color-muted
```

> Якщо колір тексту дефолтний (успадковується) — `text-color-*` не додаємо. Клас кольору ставимо лише там, де потрібно перевизначити.

> `_`-клас додаємо тексту тільки тоді, коли поряд із розміром/кольором йому потрібні власні блок-специфічні стилі (рідкісний випадок). Звичайний текст — лише токени.

---

## 7. Правила розділювачів

| Символ               | Роль                 | Приклад                        |
| -------------------- | -------------------- | ------------------------------ |
| `-` (дефіс)          | склейка слів у назві | `job-card`, `margin-top-large` |
| `_` (підкреслення)   | елемент блоку        | `job-card_header`              |
| `is-` (комбо-клас)   | варіація / стан      | `card is-featured`             |
| назва за властивістю | утиліта              | `text-align-center`            |

## Повний приклад: картка вакансії

```html
<div class="job-card is-highlighted">
  <div class="job-card_header">
    <h3 class="text-size-h4 text-color-black">Frontend Developer</h3>
    <span class="text-size-s text-color-muted">Lutsk / Remote</span>
  </div>

  <div class="job-card_tags">
    <span class="job-card_tag">Webflow</span>
    <span class="job-card_tag is-remote">Remote</span>
  </div>

  <a class="button is-primary margin-top-large">Відгукнутися</a>
</div>
```

Розбір:

- `job-card` — блок (вигляд картки), `is-highlighted` — варіація
- `job-card_header`, `job-card_tags` — layout-обгортки з власними стилями (flex, gap) → мають `_`-клас
- заголовок і локація — **чистий текст без `_`-класу**: вигляд задають `text-size-*` + `text-color-*`
- `job-card_tag` — елемент із власним виглядом чипа (фон, padding) → клас лишається; `is-remote` — варіація
- `button is-primary` — компонент + варіація
- `margin-top-large` — утиліта відступу поверх кнопки

## Чек-лист для рев'ю

- [ ] Кожна секція має базовий `section` + за потреби `is-`
- [ ] Варіації та стани оформлені через `is-`, а не новими базовими класами
- [ ] Кожен `_`-клас несе власні стилі (layout або блок-специфічний вигляд) — порожніх немає
- [ ] Текстові вузли стилізовані токенами `text-size-*` / `text-color-*` напряму, без зайвого `_`-класу
- [ ] Семантичний тег заголовка (`<h1>`…) обрано окремо від візуального `text-size-h*`
- [ ] Утиліти названі за CSS-властивістю й описують одну властивість
- [ ] Немає дублювання стилів там, де достатньо `is-`, токена або утиліти
- [ ] Назви класів читаються самодокументовано (зрозуміло без коментарів)
